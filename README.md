# PintHop 🍻

## 次の一杯を見つけよう。一人でも、友達と一緒でも。

**PintHop**は、クラフトビール体験を向上させる二つの核心機能を持つオープンソースアプリです：

1. **次の一杯を発見する**: あなたの好みや評価の高いビールを簡単に見つけ、ビール体験の質を高めます
2. **コミュニティとつながる**: 友達との自然な合流を可能にし、ビールコミュニティの知識を共有します

一人でビールを楽しむ時も、友達と共に飲む時も、PintHopはあなたのビール体験を豊かにします。事前の計画や明示的な誘い合わせなしに、今友達がどこでどんなビールを楽しんでいるかを知り、自然に合流することも、一人でじっくりと新しいビールを探索することもできます。

## 🌟 コンセプト

PintHopは二つの核心体験を融合させたアプリです：

- **次の一杯を発見する**:
  - 質の高いビールとの出会いをサポート
  - ブルワリーの得意スタイルや受賞歴の可視化
  - レビューサイト評価の一元表示
  - タップリスト情報のコミュニティ共有

- **自然なつながりを創出する**:
  - **心理的障壁ゼロ**: 「誘う/誘われる」という社会的プレッシャーなし
  - **リアルタイムプレゼンス**: 友達が今どこにいるかがすぐ分かる
  - **自然な合流**: 計画せずに偶然を装った出会いの創出
  - **プライバシー重視**: 自分の情報をどこまで共有するか完全にコントロール

一人で新しいビールを探索する喜びと、友達と偶然出会って一緒に飲む楽しさ、両方の体験を高める設計です。

## 💫 主な機能

- **ワンタップチェックイン**: ブルワリーに着いたらタップするだけ。詳細入力は必要なし
- **アンビエント存在表示**: 友達がどこにいるかを控えめに通知
- **滞在時間共有**: 「あと1時間ほどいる」などの簡易情報表示
- **プラン可視化**: 今日・今週のブルワリー訪問予定が分かる（任意共有）
- **グループダイナミクス**: 友達グループの形成・移動が自然に見える
- **リアルタイムマップ**: 現在アクティブなスポットとそこにいる友達が分かる
- **タップリスト共有**: ユーザーが最新のタップリスト写真をアップロード
- **ブルワリー詳細情報**: 得意スタイル、受賞歴、各レビューサイトの評価を表示

- **詳細なビール記録**: 
  フレーバープロファイルやホップ構成まで記録できる高度なレビュー機能

## 🚀 従来のアプリとの違い

### 1. ビール発見とコミュニティの統合
- **従来のビールアプリ**: 記録とコレクションが中心、社会的要素は二次的
- **PintHop**: ビール発見体験とリアルタイムつながりの自然な融合
- **統合価値**: 「どのビールを飲むか」と「誰と飲むか」の相乗効果創出

### 2. 「投稿」ではなく「存在」
- **従来型SNS**: 行った場所・飲んだものを後から投稿・シェア
- **PintHop**: 今どこにいるかのシンプルな共有がすべての始まり
- **記録よりも体験**: 写真・レビューは二の次、まずは体験の質を向上

### 3. 心理的ハードルの排除
- **従来型アプリ**: 「誘う→承認/拒否される」という明示的フロー
- **PintHop**: 「今ここにいます」という情報提示のみ、応答義務なし
- **自然な参加**: 「行くかも」「興味あり」など曖昧な意思表明も可能

### 4. リアルタイム体験
- **従来型SNS**: 過去の活動の記録と未来の予定の調整
- **PintHop**: 「今この瞬間」に焦点を当てた臨場感
- **即時性**: 開いた瞬間に今アクティブな場所と友達が分かる

### 5. ビール情報の統合
- **従来型ビールアプリ**: 個別のレビューやチェックインが分散
- **PintHop**: 複数レビューサイトの評価、得意スタイル、タップリストを一元表示
- **一人でも充実**: 独自のビール推薦と情報集約で一人飲みの質も向上

## 💻 技術スタック

### 現在の実装
- **フロントエンド**: React + TypeScript
- **バックエンド**: Node.js + Express
- **データベース**: MongoDB
- **リアルタイム通信**: Socket.IO
- **マップ**: Leaflet.js
- **認証**: JWT
- **ホスティング**: Netlify（フロントエンド） + rihobeer.com（バックエンド）

以前の試作段階では Firebase を用いた PWA 版を開発していましたが、
現在は Express サーバーと MongoDB を用いた構成に統一しています。

### 将来の拡張
- **プレゼンス最適化**: バッテリー負荷を抑えたリアルタイム更新
- **コンテキスト認識**: 時間帯・場所に応じた適応的挙動
- **オフライン操作**: 接続状態に関わらず利用可能な設計
- **AI機能**: タップリストOCR、ビール推薦、自然言語検索
