# PintHop アプリ仕様書（v2.1）

## 1. アプリ概要

**PintHop** は、「Find your next pint & Find your next friends」をテーマにしたクラフトビール特化型コミュニティアプリです。  
ビールの体験、コミュニティ活動、リアルタイムの共有を中心に設計され、シアトルのビールコミュニティ「シアトルビール部」を起点として展開します。

---

## 2. 主な目的

- ビールを通じた「体験」と「人とのつながり」を提供する
- リアルタイムで誰がどこで何を飲んでいるかを把握できる
- その場に参加するためのきっかけを提供する
- 知識と経験豊富なビール愛好家とのつながりが作れる
- テーマ性のあるビアホッピングを地図上で簡単に設計・共有できる
- 各ブルワリーの特徴が一目で分かる
- ビール資格保持者の知識を共有できるプラットフォームを提供する
- ローカルコミュニティ・イベントの発見と参加を促進する

---

## 3. 主な機能（MVP）

| 機能カテゴリ         | 内容                                                                 |
|----------------------|----------------------------------------------------------------------|
| 超簡易記録           | ワンタップチェックイン、写真アップロード（タップリスト/外観/ビール）、最小限の入力項目 |
| コミュニティ活動     | サークル機能、活動予定表示、参加登録、過去活動レポート閲覧、メンバー間メッセージ |
| リアルタイム共有     | 現在飲んでいるビール、場所のチェックイン機能、「今から参加OK」フラグ設定、参加表明機能 |
| ビアホッピング体験   | マップベースのテーマ別ブルワリーツアー設計、所要時間や難易度表示、ナビゲーション |
| 写真データベース     | タップリスト/外観/ビール写真の自動分類保存、写真検索、時系列表示 |
| コミュニティレビュー | シンプルな評価機能（いいね/星評価のみ）、コメントオプション |
| バッジシステム       | 訪問実績、テイスティング経験、イベント参加などに応じたバッジ付与、表示機能 |
| 資格登録システム     | BJCP、Cicerone、SSI（日本）などのビール関連資格の登録、認証、専門家バッジ表示 |
| 位置情報検索         | 現在地または指定位置からの半径距離（0.5km/1km/5km/10km）でブルワリー検索 |
| 近接性ソート         | 現在地からの距離順にブルワリーをソート表示。徒歩/自転車/車での所要時間表示 |
| 地図表示             | 現在地 or 指定都市周辺のブルワリー位置、「今飲んでいる人」の表示 |
| クラフトビール知識   | ビアスタイルガイド、テイスティング用語解説、「今日のビール豆知識」機能 |
| イベント管理         | ユーザー作成イベント、サークルイベント、地域イベントの統合表示・管理 |
| フォロー機能         | ユーザー間のフォロー・フォロワー関係、アクティビティフィード表示 |

## 4. コミュニティ機能の詳細

### サークル機能

- **オープンサークル設計**：
  - 誰でも作成可能な「サークル」（ビール愛好会、テーマ別グループなど）
  - 公開/招待制/承認制の3種類のプライバシー設定
  - サークル単位でのイベント作成・管理
  - サークルメンバー間の専用メッセージ機能
  - サークル活動のカレンダー表示・通知

- **サークル管理**：
  - サークル管理者権限（イベント承認、メンバー管理など）
  - サークル活動レポート（訪問ブルワリー、飲んだビールなど）
  - メンバー達成バッジの共有・表示
  - サークルページカスタマイズ（ヘッダー画像、説明文など）
  - サークル内ランキング（訪問数、チェックイン数など）

- **シアトルビール部統合**：
  - シンプルな部活動カレンダー表示
  - 「参加」ボタンをタップするだけで登録完了
  - 過去イベントの写真ギャラリー簡単閲覧
  - スマホのカレンダーアプリへのワンタップ連携
  - サークル専用のアクティビティフィード

### 超簡易リアルタイム共有

- **ワンタップチェックイン**：
  - アプリを開くと「近くのブルワリー」が自動表示
  - ブルワリーをタップするだけでチェックイン完了
  - ビール選択・コメントは完全に任意
  - 「写真を撮る」ボタンでの直接カメラ起動

- **最小限の参加障壁**：
  - 「今から行く」ボタン一つで参加表明
  - 自己紹介等の入力なしで即参加可能
  - プッシュ通知でリアルタイムに友人の動きを確認
  - 招待は写真付きで一目で分かりやすく

- **シンプルなアクティブスポット表示**：
  - 地図上に人数のみ表示（詳細は不要）
  - 多くの人がいる場所は目立つマーカーで表示
  - 「今日のホットスポット」として自動ハイライト
  - タップすると現在のタップリスト写真を表示（利用可能な場合）

### ユーザーフォロー機能

- **シンプルなフォロー機構**：
  - ユーザープロフィールからワンタップでフォロー/アンフォロー
  - フォロー中/フォロワー一覧表示
  - 新規フォロワー通知（任意設定）
  - フォロー関係に基づく推奨ユーザー表示

- **フォローフィード**：
  - フォロー中ユーザーのチェックイン活動表示
  - ビアホッピングプラン共有時の優先表示
  - フォロー中ユーザーのイベント参加情報表示
  - 「フォロー中の友達も行きました」通知

- **プライバシー設定**：
  - フォロー可否の設定（全員/フレンドのみ/非公開）
  - アクティビティ公開範囲設定（全員/フォロワーのみ/フレンドのみ）
  - 位置情報共有設定（全員/フォロワーのみ/フレンドのみ/非公開）
  - チェックイン公開範囲の個別設定

### フォトベースバッジシステム

- **写真アップロード達成バッジ**：
  - タップリスト写真10枚達成
  - ビール写真50枚達成
  - ブルワリー外観写真20枚達成
  - 写真コレクション完成（特定ブルワリーの3種類全て）

- **シンプル訪問バッジ**：
  - 訪問数だけでカウント（5/10/20/50/100軒）
  - エリア制覇（特定エリアの全ブルワリー訪問）
  - 週末ブルーワー（週末の連続訪問）
  - 先駆者（新しいブルワリーへの初訪問者）

- **コミュニティバッジ**：
  - 情報提供数（タップリスト更新など）
  - 「ありがとう」獲得数
  - 招待成立数
  - イベント参加数

- **資格バッジ**：
  - BJCP（ビールジャッジ認定プログラム）保持者
  - Cicerone（ビアサーブ資格）レベル別
  - 醸造関連資格
  - 各国・地域のビール専門資格

## 5. 「体験」重視の設計詳細

### 超簡易記録システム

- **最小限の入力フォーム**：
  - 場所のみ選択でチェックイン完了（GPSから自動選択）
  - 写真アップロードのみでも投稿可能
  - スワイプ操作での評価（右=良い、左=イマイチ）
  - 任意テキスト入力（必須項目なし）
  - 事前データ入力なしの写真保存機能

- **写真カテゴリ分類**：
  - タップリスト写真（自動タグ付け）
  - 外観写真（ブルワリー/バーの外観・内装）
  - ビール写真（グラスに注がれたビール）
  - タグ付けはアップロード時にワンタップで選択可能
  - 写真からのブルワリー情報自動抽出（将来機能）

- **ブルワリー情報のクラウドソーシング**：
  - ユーザーによるタップリスト写真の共有
  - 写真のみでの情報更新（入力テキストなしでOK）
  - 最新のタップリストを自動でトップに表示
  - タップリスト写真の更新日時と投稿者表示
  - 「最新情報をありがとう」ボタン

### 詳細記入オプション（任意）

- **詳細なレビュー機能**：
  - 「もっと書く」ボタンでの詳細入力画面展開
  - 詳細テイスティングノートのテンプレート提供
  - 記入済み項目の保存と再利用
  - マイ評価テンプレートの作成と共有
  - 書けば書くほど獲得バッジ/ポイントが増加

- **体験ストーリー記録**：
  - 写真に紐づく「思い出メモ」機能
  - 「その日の体験」タイムライン自動生成
  - 同行者タグ付けとメモリー共有機能
  - のちの「振り返り機能」でサプライズ表示
  - 「1年前の今日」リマインド機能

- **ビールマスターノート**：
  - 専門家向け詳細評価フォーム（任意）
  - BJCPスタイルガイドライン準拠の評価項目
  - 資格保持者専用の詳細評価機能
  - 評価履歴の時系列比較分析
  - Tasting Masterバッジ進行状況表示

### テーマ別ビアホッピング（マップベース）

- **マップベース設計**：
  - 視覚的に理解しやすいマップインターフェース
  - ルート表示はシンプルな線と番号のみ（視覚的煩雑さを回避）
  - 各ポイントをタップで簡易ブルワリー情報を表示
  - 3Dビューオプション（地形に合わせたルート表示）
  - ピンチ・ズームでの直感的操作

- **ストーリーベース設計**：
  - 「IPAの歴史を辿る旅」（伝統的IPAから最新NEIPAまで）
  - 「シアトルのビール革命者たち」（地域のパイオニアブルワリーを巡る）
  - 「一日で世界一周」（各国スタイルを巡る）
  - 「地元食材の魅力」（地域特産品を活用するブルワリー巡り）

- **難易度と種類**：
  - 初心者向け（定番ビールと基本スタイル）
  - マニア向け（レア・個性的ビール）
  - 所要時間別（2時間コース、半日コース、週末コース）
  - 交通手段別（徒歩、自転車、公共交通機関、車）

- **カスタマイズとシェア**：
  - 自分だけのテーマルート作成（マップ上でドラッグ&ドロップ）
  - SNSやサークル内でのルート共有
  - 実際に巡った記録と写真のフォトジャーナル作成
  - コミュニティでの人気ルートランキング

- **ナビゲーション機能**：
  - マップアプリ連携（Google Maps、Apple Maps）
  - ターン・バイ・ターン音声ナビゲーション
  - 公共交通機関情報表示
  - 予測所要時間とリアルタイム調整
  - オフラインマップダウンロードオプション

### ワンタップチェックイン

- **超簡易チェックイン**：
  - アプリを開いたらトップに現在地近くのブルワリーを表示
  - ブルワリー名をタップするだけでチェックイン完了
  - 追加情報は任意（後からでも追加可能）
  - 「また来た」ボタンでリピート訪問を簡単記録
  - 「友達を誘う」ワンタップボタン

- **写真優先の記録**：
  - ビール写真を撮って即アップロード
  - タップリスト写真の簡易共有
  - 後からタグ付けやコメント追加オプション
  - 写真の一括アップロード機能（フライト対応）
  - 訪問時の自動リマインド「タップリスト写真をアップしませんか？」

- **超簡易評価**：
  - 「👍」または「👎」のみの二択評価オプション
  - 星評価（必要な場合のみ任意で追加）
  - 「また飲みたい」チェックボックス
  - プリセットコメント選択（「フルーティ」「ホッピー」など）
  - カスタム絵文字リアクション

### 教育的コンテンツ

- **ビアスタイルガイド**：
  - BJCPスタイルガイドラインの解説
  - スタイルの歴史と特徴
  - 地域別の典型的なスタイル紹介
  - 食事とのペアリング提案

- **テイスティング講座**：
  - ステップバイステップの味わい方
  - オフフレーバーの識別方法
  - 資格保持者によるプロのコツ
  - バーチャルテイスティングイベント

- **ブルワリー知識**：
  - 醸造工程の解説
  - 原料の違いによる味わいの変化
  - ブルワリーごとの特徴的な製法
  - 季節・トレンドによる製品の変化

## 6. レビュー・評価システム

### 心理学に基づく投稿促進設計

- **即時フィードバック**：
  - 写真投稿時の即時「いいね」効果音とアニメーション
  - 「あなたの写真がN人に見られました」リアルタイム通知
  - 「あなたのお陰で10人がこのブルワリーに来ました」効果測定
  - 投稿後30分以内の反応を優先表示（即時満足感）

- **社会的証明効果**：
  - 「友達もここで写真を撮っています」通知
  - 「あなたの写真で○○さんが行きたいと言っています」メッセージ
  - 「あなたの貢献度：シアトルTOP10%」ステータス表示
  - フレンドのアクティビティを目立つ位置に表示

- **達成感の演出**：
  - 少ない投稿でも価値ある貢献として表示
  - コレクション完成度の視覚的表示（「あと1枚でコンプリート！」）
  - 予期せぬ「サプライズバッジ」の付与
  - マイルストーン達成祝福アニメーション

- **適切な難易度設計**：
  - 初回投稿は超簡単に（タップ1回で完了）
  - 徐々に詳細情報のオプションを提示
  - 投稿回数に応じた「次のレベル」チャレンジ
  - 「ビールマスター」向け詳細記入オプション（自己表現欲求対応）

### 超シンプルなレビュー＆詳細オプション

- **ワンタップ評価**：
  - 👍/👎のみの二択評価
  - 5段階星評価（任意）
  - ワンタップ絵文字リアクション（🍺🔥❄️etc.）
  - プリセットコメント選択（タップで追加）
  - 自由コメントは完全任意

- **100点満点評価システム**：
  - スライダーで直感的に100点満点評価
  - 数値入力も可能（キーボード入力）
  - 評価時の感情表情アイコン連動
  - 自分だけの評価分布グラフ自動生成
  - 評価の時系列変化トラッキング（成長記録）

- **写真ベースの評価**：
  - 写真のみのアップロードでもOK
  - 写真へのリアクション機能
  - 写真でのタップリスト共有
  - 「ビールフォトギャラリー」自動生成
  - 写真による簡易的なビール記録

- **集計・分析**：
  - 簡易集計（「みんなの評価」）
  - ビール・ブルワリー別の「いいね率」
  - 最も写真がアップロードされたビール
  - 「また飲みたい」登録数ランキング
  - 時間帯別のチェックイン人気マップ

### 外部レビューサイト情報の手動管理

- **レビューサイト統合データベース**：
  - Untappd、RateBeer、BeerAdvocate、The Beer Connoisseurなどの外部サイトスコアを記録
  - ビール・ブルワリーごとに各サイトの評価スコアを統合表示
  - 各サイトの評価を5点満点で正規化して表示
  - 時系列での評価変動追跡機能

- **超簡易データ入力**：
  - スワイプによる数値入力
  - 「最近見た」サイトの評価をワンタップ入力
  - サイト名と数値のみ（コメントなどの詳細情報は任意）
  - 写真アップロードによる情報共有（スクリーンショット等）
  - 自動保存（確認画面なし）

- **管理者更新システム**：
  - 管理者による主要ビール・ブルワリー情報の定期的な手動更新
  - 更新履歴とデータ鮮度の表示
  - 管理者に対する更新が必要なアイテムの自動通知
  - 更新プライオリティキューの管理（人気順、リクエスト順など）

- **ユーザー貢献システム**：
  - タップリストの写真アップロードによる情報更新
  - その場で撮影 → 即アップロードのフロー
  - 更新はアップロードだけで完了（承認フローなし）
  - 信頼度の高いユーザーの情報を優先表示
  - 「最新情報ありがとう」ボタン

### シンプルな推奨機能

- **基本的な推奨機能**：
  - 近くの人気ブルワリー
  - 「みんながよく飲むビール」
  - 「初めての人におすすめ」リスト
  - 時間帯別のおすすめスポット
  - 今最もアクティブなスポット

- **友人ベースの推奨**：
  - 友達が最近飲んだビール
  - 友達がよく行くブルワリー
  - 「友達と一緒に行ったところ」リスト
  - 「友達からの招待」機能
  - 「一緒に飲みたい」ボタン

## 7. ブルワリー情報管理

### ブルワリープロフィール

- **基本情報表示**：
  - 名前、住所、連絡先
  - 営業時間とリアルタイム営業状況
  - 外観・内装の写真ギャラリー
  - 最新のタップリスト（ユーザー提供写真）
  - 現在のチェックイン人数

- **ユーザー評価情報**：
  - 100点満点での総合評価スコア
  - 「いいね」率と「また来たい」登録数
  - 最新の評価コメント
  - 人気の曜日・時間帯（訪問データ分析）
  - 評価の時系列推移グラフ

- **ビアスタイル情報**：
  - ブルワリーの得意ビアスタイルタグ表示
  - ユーザー投票による人気ビアスタイルランキング
  - 提供ビアスタイルの多様性スコア
  - シーズナル特化スタイルカレンダー
  - スタイル別の評価比較

### ブルワリー得意スタイル登録システム

- **ワンタップスタイル登録**：
  - 「このブルワリーの得意スタイルを教えて」プロンプト
  - 既存スタイルのサジェスト機能（人気順表示）
  - タップするだけで登録完了（複数選択可）
  - 登録後に「ありがとう！」アニメーション表示
  - 任意のスキップオプション（「今はしない」）

- **自由スタイル追加**：
  - 「+ 新しいスタイルを追加」ボタン
  - 自由入力フィールド（予測変換サポート）
  - 類似スタイルのサジェスト機能
  - BJCP準拠スタイル一覧からの選択オプション
  - 独自スタイル名の登録も可能

- **コミュニティベースのスタイル投票**：
  - 各ブルワリーの「得意スタイル」にユーザー投票

## 12. 認証システム設計

### 認証フロー

- **ソーシャルログイン優先アプローチ**：
  - Googleアカウント連携を主要ログインオプションとして提供
  - Apple IDによるログインをiOSユーザー向けに提供 
  - 専用アカウント作成は不要（ソーシャルログインのみでアクセス可能）
  - 年齢確認プロセスをログインフロー内に統合

- **初回ログインフロー**：
  - Google/Apple認証完了後、最小限のプロフィール設定画面に遷移
  - ニックネーム入力（デフォルトはGoogleアカウント名）
  - プロフィール写真選択（デフォルトはGoogleアカウント画像）
  - 好みのビアスタイル選択（任意、スキップ可能）
  - 年齢確認（必須、地域ごとの法的要件に対応）

- **再ログインフロー**：
  - アプリ起動時に自動ログイン（認証情報保持）
  - 認証トークン期限切れ時の自動リフレッシュ
  - デバイス変更時の簡易再認証
  - 長期不使用後の簡易再認証

### ユーザー情報管理

- **取得する情報**：
  - 基本情報：名前、メールアドレス、プロフィール画像（ソーシャルアカウントから取得）
  - アプリ固有情報：好みのビアスタイル、バッジ、チェックイン履歴など（アプリ内で生成・保存）

- **プライバシー設定**：
  - メールアドレスの用途明示（必要最小限の通知のみ）
  - ソーシャルアカウントからの追加情報取得は行わない方針
  - プロフィール情報の公開範囲設定（デフォルトはフレンドのみ）
  - 位置情報共有の詳細設定オプション

### 技術実装

- **Firebase Authentication活用**：
  - GoogleおよびApple IDプロバイダーの設定
  - OAuth 2.0フローの実装
  - セキュアなトークン管理
  - クロスプラットフォーム対応

- **セキュリティ対策**：
  - アクセストークンのセキュアな保存
  - リフレッシュトークンの適切な管理
  - サーバーサイド検証の徹底
  - 不正アクセス検知と防止機能

### 「超簡易」原則の適用

- **ユーザー体験最適化**：
  - ログイン画面でのボタン2つのみの表示（「Googleでログイン」「Appleでログイン」）
  - アカウント作成とログインの区別をなくす（どちらも同じボタン）
  - 必要最小限の情報入力でのアプリ利用開始
  - プロフィール詳細情報は後から任意で追加可能な設計

- **エラーハンドリング**：
  - 認証エラー時のわかりやすいガイダンス
  - インターネット接続問題の検出と対応
  - 認証復旧の自動再試行
  - オフラインモードへの自動切替オプション

## 18. 地域別ブルワリー情報管理システム

全米規模での拡張性を考慮した地域別データ管理システムを実装します。データベースをメインストレージとしながら、JSONファイルによる柔軟な更新も可能にするハイブリッドアプローチを採用します。

### 地域ベースのデータ構造

- **階層型地域構造**：
  - 国レベル（アメリカ全土）
  - 地域レベル（西部、中西部、東部、南部）
  - 州レベル（ワシントン州、オレゴン州など）
  - 都市レベル（シアトル、ポートランドなど）
  - 地区レベル（任意）

- **地域IDの設計**：
  - 地域識別コード体系（例：`US-W-WA-SEA`）
  - 階層型クエリのサポート（親地域に含まれるすべての子地域を取得可能）
  - 検索最適化インデックス

### JSONファイルベースの管理システム

- **地域別JSONファイル構造**：
  ```json
  {

    "lastUpdated": "2025-04-13T08:00:00Z",
  ```

- **JSONファイルのエクスポート機能**：
  - 地域単位でのデータエクスポート
  - 指定した地域のブルワリー情報を最新のJSONファイルとして出力
  - 管理用のタイムスタンプとバージョン情報の付与
  - 差分エクスポートオプション（前回から変更された情報のみ）

- **JSONファイルのインポート機能**：
  - 地域単位でのデータインポート
  - バリデーションチェック（地域IDの一致確認など）
  - 既存データとの競合解決（タイムスタンプベース）
  - 一括または選択的なデータ更新オプション
  - 更新履歴の自動記録

### 管理者向けインターフェース

- **地域管理者権限システム**：
  - 地域ごとの管理者割り当て
  - 管理者の権限レベル設定（読み取り専用/編集可能/承認権限など）
  - 親地域管理者は子地域データにアクセス可能

- **データ管理ダッシュボード**：
  - 地域別データの概要表示
  - 更新が必要なブルワリーの優先表示
  - ユーザー貢献状況のモニタリング
  - データ品質スコアの表示

- **JSONファイル操作UI**：
  - ファイルアップロード/ダウンロードインターフェース
  - インポート前の差分プレビュー
  - バリデーションエラーの視覚的表示
  - インポート成功/失敗レポート

- **更新履歴管理**：
  - 更新履歴の詳細表示
  - 更新者、日時、変更内容の記録
  - 変更のロールバック機能
  - 履歴比較ビュー

### 拡張性対応の実装戦略

- **スケーラブルなデータベース設計**：
  - 地域ベースのシャーディング
  - 読み取り専用レプリカの地域分散
  - キャッシュ層の最適化（人気地域優先）

- **段階的データ移行**：
  - 初期フェーズ（シアトル）: 単一JSONファイル管理
  - 中期フェーズ（州レベル）: 地域別JSONファイル分割
  - 大規模フェーズ（全米）: 完全な地域分散データベース

- **地域間データ同期**：
  - 複数地域にまたがるブルワリーチェーンの一貫性確保
  - 地域データの定期的な整合性チェック
  - クロスリージョン検索の最適化

### ユーザー貢献と地域データの統合

- **地域ベースの貢献フロー**：
  - ユーザー貢献は自動的に適切な地域に割り当て
  - 地域管理者による貢献レビュー（必要に応じて）
  - 高信頼度ユーザーの貢献は自動承認

- **地域データの統計とインサイト**：
  - 地域別のビアスタイル傾向分析
  - 地域間の人気ブルワリー比較
  - 地域特性の自動抽出（「IPA特化地域」など）
